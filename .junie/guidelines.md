# Daily Report Generation Guidelines

These guidelines describe the process for adding a new daily report (French and English) based on current AI news, trends, tools, and personalities.

## 1. File Structure and Naming

- Generate two files for the current date (format: `YYYYMMDD.md`):
    - French version: `daily/fr/YYYYMMDD.md`
    - English version: `daily/en/YYYYMMDD.md`

## 2. Frontmatter Requirements

Each report MUST include the following frontmatter:

```yaml
---
layout: daily
title: "AI News - Month Day, Year" (English) or "Actualités IA - Day Month Year" (French)
date: YYYY-MM-DD
news_summary: "Brief summary of key news items"
resource: "Name of the resource of the day"
tool: "Name of the tool of the day"
person: "Name of the AI personality of the day"
technology: "Name of the key technology/concept of the day"
---
```

## 3. Content Sections

### Cross-Linking
At the top of each file, add a link to the other language version:
- In French file: `[View in English]({{ site.baseurl }}/daily/en/YYYYMMDD.html)`
- In English file: `[Version Française]({{ site.baseurl }}/daily/fr/YYYYMMDD.html)`

### Chuck Norris Section
Include a Chuck Norris quote about AI using this HTML structure:
```html
<div class="chuck-norris-section">
    <img src="https://upload.wikimedia.org/wikipedia/commons/3/30/Chuck_Norris_May_2015.jpg" alt="Chuck Norris" class="chuck-norris-img">
    <div class="chuck-norris-content">
        <div class="chuck-norris-quote">
            "INSERT_QUOTE_HERE"
        </div>
        <div class="chuck-norris-footer">
            <span class="chuck-norris-rip">RIP MESSAGE_HERE</span>
        </div>
    </div>
</div>
```

### News Items
- At most 3 important news items.
- All news items should talk about a new topic, not already mentioned in a previous daily report, except if it is a new version or a significant update.
- Title, 2 paragraphs of description, and a link to the detailed description with the date of the original news.
- Use a list item for the link: `- **Link:** [Title](URL) (Month Day, Year)` (English) or `- **Lien :** [Titre](URL) (Day Month Year)` (French)
- For each item (except Chuck Norris), indicate if it is **Stable** or **Volatile**.

### Trend
- One AI trend with a 3-paragraph description.
- The trend should be a new topic, not already presented in a previous daily report.
- Use a list item for the link if applicable: `- **Related Resource:** [Title](URL)`
- Indicate if it is **Stable** or **Volatile**.

### Resource
- Presentation of one resource (name, description).
- The resource should be a new topic, not already presented in a previous daily report.
- Use a list item for the link: `- **Resource:** [Title](URL)`
- Indicate if it is **Stable** or **Volatile**.

### Tool, Person, and Technology
For these three items, you MUST:
1. Ensure they are new topics not already presented in a previous daily report, except for new versions of a Tool or Technology.
2. Create/Update a file in the `_technologies/` folder (see Section 4).
3. Present them in the daily report with links to their technical sheets.
4. For Tool and Technology, use a list item for the links: `- **Links:** [Website](URL) • [Technical Sheet]({{ site.baseurl }}/technologies/SLUG/)`

#### AI Personality Section Structure
- The presentation of one key person of the AI, not already presented in a previous daily report, with a real photo of the person and a description of their personality.
- Indicate if it is **Stable** or **Volatile**.
- Use the following HTML structure.
```html
<div class="ai-personality-section">
    <img src="IMAGE_URL" alt="NAME" class="ai-personality-img">
    <div class="ai-personality-content">
        <span class="ai-personality-name">NAME</span>
        <p class="ai-personality-description">DESCRIPTION</p>
        <div class="ai-personality-footer">
            <div class="ai-personality-links">
                <a href="MAIN_LINK">Main Resource</a> | <a href="{{ site.baseurl }}/technologies/SLUG/">Technical Sheet</a>
            </div>
            <span class="ai-personality-status">Status/Statut : Stable</span>
        </div>
    </div>
</div>
```

### Disclaimer
Add at the bottom of the page:
- English: `*Disclaimer: This page was generated by AI.*`
- French: `*Avertissement : Cette page a été générée par une IA.*`

## 4. Required File Updates

### `_technologies/` Folder
- For any new **Tool**, **Person**, or **Technology** introduced, create a new Markdown file in `_technologies/`.
- Prioritize using the `picture` URL if it exists in previous entries.
- Ensure the slug used in the daily report matches the filename in `_technologies/`.

### `models.md`
- If a news item is about a new **AI Model version**:
    - Open `models.md`.
    - Find the corresponding series (e.g., GPT Series, Claude Series).
    - Add a new `version-item` entry at the top of the history for that series.
    - Include the version name, date, a short description, and a link to the **English** version of the generated daily report.

### `index.html`
- Update the "Read AI News" highlight on the homepage to link to the new **English** daily report.
- Example: Update line 12 to `href="{{ '/daily/en/YYYYMMDD.html' | relative_url }}"` and line 13 to the current date.

## 5. Verification Checklist
- [ ] Both `fr` and `en` files created.
- [ ] `_technologies/` files created for new items.
- [ ] `models.md` updated if new model versions were mentioned.
- [ ] `index.html` updated to point to the new English report.
- [ ] Cross-links between language versions are correct.
- [ ] HTML structures for Chuck Norris and AI Personality are properly used.
